# Phase 1 — UI Design Contract

> Visual and interaction contract for BarraBrava Phase 1: Foundation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Platform: Mobile (iOS + Android). Godot 4.3. Canvas design target: 1080x1920px. Min viewport width: 360px.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Godot Theme Resources (.tres) — one shared ThemeResource applied project-wide |
| Preset | Not applicable (Godot native theming) |
| Component library | Godot Control node scenes (.tscn) — reusable scene instancing pattern |
| Icon library | None in Phase 1 — use Unicode glyphs or simple ColorRect shapes. SVG icons deferred to Phase 2+ |
| Font | Nunito (Google Fonts, OFL 1.1 license) — single family, two weights: Regular 400 + Bold 700 |

**Visual register:** Bold arcade — flat colors, high contrast, clean shapes, vibrant. Reference tone: FIFA Mobile / Clash of Clans. Optimized for readability on low-end screens.

## Spacing Scale

Declared values — all multiples of 4. Applied via Godot `separation`, `margin_*`, `custom_minimum_size`, and `offset_*` properties on Control nodes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, chip internal padding |
| sm | 8px | Compact element spacing, inner card padding |
| md | 16px | Default element spacing, button horizontal padding |
| lg | 24px | Section padding, card-to-card gaps |
| xl | 32px | Layout gaps between major UI blocks |
| 2xl | 48px | Major section breaks, screen-level top padding |
| 3xl | 64px | Page-level vertical rhythm, bottom safe-area buffer |

**Exceptions:**
- Touch targets: minimum 44px height for all interactive Control nodes (Button, OptionButton, LineEdit) — enforced via `custom_minimum_size.y = 44`. This overrides the 8-point grid where needed.
- iOS notch / Android status bar: top safe-area inset read from `DisplayServer.get_display_safe_area()` at runtime. Add inset as top margin to root CanvasLayer. Not a fixed token — runtime value.
- Bottom safe-area (iOS home indicator): minimum 34px additional bottom margin on HomeScreen nav bar.

## Typography

Font family: **Nunito** loaded as two `FontFile` resources in the shared Theme:
- `res://assets/fonts/Nunito-Regular.ttf` — weight 400
- `res://assets/fonts/Nunito-Bold.ttf` — weight 700

All sizes in Godot `theme_override_font_sizes` — unit is screen pixels at 1080px canvas width. Godot scales proportionally via `stretch_mode = canvas_items`.

| Role | Size | Weight | Line Height | Godot Node |
|------|------|--------|-------------|------------|
| Display | 28px | Bold (700) | 1.1 | Label with theme override — club name, pibe name |
| Heading | 22px | Bold (700) | 1.2 | Label — screen titles, section headers |
| Label / Button | 14px | Bold (700) | 1.2 | Button text, chip labels, tab labels |
| Body | 16px | Regular (400) | 1.5 | RichTextLabel — descriptions, tutorial copy, empty states |

**Rules:**
- Never mix weights within a single Label node — use separate nodes if a line has mixed weight.
- Godot does not support CSS `line-height` natively. Simulate with `autowrap_mode` + container sizing. For RichTextLabel body text, set `fit_content = true` and control spacing via VBoxContainer separation.
- No italic, no underline, no strikethrough in Phase 1.

## Color

All colors defined as `Color` constants in `res://autoload/Theme.gd` (or as StyleBox overrides in the shared `.tres`).

| Role | Hex | Godot Color | Usage |
|------|-----|------------|-------|
| Dominant (60%) | `#1A1A1A` | `Color(0.102, 0.102, 0.102, 1)` | Main background (ColorRect), screen backgrounds, primary surfaces |
| Secondary (30%) | `#2D2D2D` | `Color(0.176, 0.176, 0.176, 1)` | Cards (PanelContainer), containers, nav bar background |
| Accent (10%) | `#D62828` | `Color(0.839, 0.157, 0.157, 1)` | Reserved list below |
| Destructive | `#E67E22` | `Color(0.902, 0.494, 0.133, 1)` | Destructive actions only (see below) |
| Text primary | `#F5F5F5` | `Color(0.961, 0.961, 0.961, 1)` | All primary labels, headings, button text |
| Text secondary | `#A0A0A0` | `Color(0.627, 0.627, 0.627, 1)` | Muted labels, helper text, placeholder text |

**Accent `#D62828` reserved exclusively for:**
1. Primary CTA button background (one per screen)
2. Active tab / active state indicator underline or highlight
3. Match window open indicator dot / badge
4. Selected club card border in ClubPickerScreen
5. Focus ring on currently focused LineEdit (AuthScreen, PibeCreatorScreen)

**Destructive `#E67E22` reserved exclusively for:**
1. Any action that deletes or resets data (Phase 1 has none — reserved for Phase 2+ account deletion)
2. Distinct from accent to prevent confusion: orange vs. red, visually unambiguous

**Color rules:**
- No gradients in Phase 1. Flat fills only.
- No transparency / alpha on text — always full opacity on its background.
- Disabled state: apply 40% opacity (`modulate.a = 0.4`) to the entire Control node, not to individual colors.

## Interaction Contract

**Platform:** Touch-only. No hover states. No cursor changes.

| Interaction | Godot Signal / Method | Visual Feedback |
|-------------|----------------------|----------------|
| Tap button | `pressed` signal | `scale` tween: 0.95 for 80ms, return 1.0 for 80ms (subtle press feel) |
| Tap list item (club) | `gui_input` ItemList or custom Control | Background flash to `#3D3D3D` for 100ms |
| Text input focus | `focus_entered` signal | Border color changes to `#D62828` (accent), 2px StyleBoxFlat border |
| Text input unfocus | `focus_exited` signal | Border returns to `#3D3D3D` |
| Screen transition | `SceneTree.change_scene_to_file` | Fade-out 150ms → fade-in 150ms via AnimationPlayer on CanvasLayer |
| Pull-to-refresh | Not in Phase 1 | N/A |
| Swipe navigation | Not in Phase 1 | N/A |

**No long-press interactions in Phase 1.**

## Component Inventory

Each component is a reusable `.tscn` scene instantiated where needed. All inherit from Control.

### 1. SplashScreen.tscn

**Purpose:** First-launch disclaimer + loading progress.

**Layout:** Full-screen ColorRect (`#1A1A1A`) → VBoxContainer centered vertically.

**Elements:**
- Game logo (Label "BarraBrava" — Display 28px Bold `#D62828`, or placeholder ColorRect if art not ready)
- Disclaimer Label (Body 16px Regular `#A0A0A0`): "Esto es una parodia de fútbol argentino. Cualquier parecido con personas reales es coincidencia."
- ProgressBar (custom StyleBox: track `#2D2D2D`, fill `#D62828`, height 4px)
- Loading label (Label 14px Bold `#A0A0A0`): "Cargando..." — hidden once complete

**Safe area:** Top margin = safe area inset from `DisplayServer.get_display_safe_area()`.

**Transitions out:** Fade to AuthScreen after loading completes.

---

### 2. AuthScreen.tscn

**Purpose:** Login + registro tabs (email + password).

**Layout:** Full-screen `#1A1A1A` → VBoxContainer with top padding 48px (2xl).

**Structure:**
```
VBoxContainer
  └─ Label (Heading 22px Bold #F5F5F5): "BarraBrava"
  └─ TabContainer (tabs: "Entrar" | "Registrarse")
       └─ [Login tab]
            └─ VBoxContainer (spacing 16px)
                 └─ LineEdit — email (placeholder "tu@correo.com", text #F5F5F5, bg #2D2D2D)
                 └─ LineEdit — password (secret=true, placeholder "contraseña")
                 └─ Button (primary CTA): "Entrá"
                 └─ Label (error, hidden by default, Body 16px #E67E22)
       └─ [Registro tab]
            └─ VBoxContainer (spacing 16px)
                 └─ LineEdit — email
                 └─ LineEdit — password
                 └─ LineEdit — confirm password
                 └─ RichTextLabel (Body 14px #A0A0A0): "Antes de arrancar, leé las reglas del juego" (bbcode link → privacy policy URL)
                 └─ Button (primary CTA): "Armar mi cuenta"
                 └─ Label (error, hidden by default)
```

**Empty state (no account yet, on login tab):**
Label below tab: "Todavía no estás en la vuelta. Anotate." (Body 16px `#A0A0A0`) — shown only on first launch before any session.

**Error states:**
- Wrong password: "Código equivocado, chabón. Probá de vuelta." (Body 16px `#E67E22`, shown inline below password field)
- Network error: "Sin conexión, chabón. Fijate el WiFi." (Body 16px `#E67E22`)
- Email already registered: "Ese mail ya está en la vuelta. Entrá con tu cuenta." (Body 16px `#E67E22`)

**LineEdit styling (StyleBoxFlat):**
- Background: `#2D2D2D`
- Border: 2px `#3D3D3D` default, `#D62828` on focus
- Corner radius: 8px
- Padding: 12px horizontal, 12px vertical
- Minimum height: 44px (touch target)
- Font: Body 16px Regular `#F5F5F5`
- Placeholder font color: `#A0A0A0`

**Primary CTA Button styling (StyleBoxFlat):**
- Background: `#D62828`
- Border: none
- Corner radius: 8px
- Padding: 16px horizontal, 12px vertical
- Minimum height: 44px
- Font: Label 14px Bold `#F5F5F5`
- Disabled state: `modulate.a = 0.4`
- Press animation: scale tween 0.95 → 1.0

### 3. ClubPickerScreen.tscn

**Purpose:** Scrollable club list with search and division filter.

**Layout:** Full-screen `#1A1A1A` → VBoxContainer.

**Structure:**
```
VBoxContainer
  └─ Label (Heading 22px Bold #F5F5F5): "Elegí tu club"
  └─ LineEdit — search bar (placeholder "Buscar club...", icon-free in Phase 1)
  └─ HBoxContainer — division filter chips (scrollable horizontal)
       └─ ChipButton × N (divisions: "Primera" | "Nacional" | "B Metro" | "Federal A" | "C Metro" | "Todos")
  └─ ScrollContainer
       └─ VBoxContainer — club list
            └─ ClubCard.tscn × N
  └─ Button (primary CTA, disabled until selection): "Me banco este club"
```

**ChipButton (custom scene):**
- Unselected: bg `#2D2D2D`, border 1px `#3D3D3D`, text 14px Bold `#A0A0A0`, corner radius 16px, padding 8px horizontal 8px vertical, min height 32px
- Selected: bg `#D62828`, border none, text 14px Bold `#F5F5F5`
- Min touch width: 64px

**ClubCard.tscn:**
- PanelContainer, bg `#2D2D2D`, corner radius 8px, padding 16px
- Row: [ColorRect 40x40 placeholder crest] + [VBoxContainer: Label club name 16px Bold `#F5F5F5` + Label division 14px Regular `#A0A0A0`]
- Selected state: 2px border `#D62828`
- Min height: 64px (touch target satisfied by card height)

**Empty state (no search results):**
- VBoxContainer centered in ScrollContainer
- Label (Heading 22px Bold `#F5F5F5`): "Nada por acá."
- Label (Body 16px Regular `#A0A0A0`): "Ese club no está en la liga. Buscá de nuevo."

**CTA disabled state:** `modulate.a = 0.4` until one ClubCard is selected.

### 4. PibeCreatorScreen.tscn

**Purpose:** Name input + fixed base stats display.

**Layout:** Full-screen `#1A1A1A` → VBoxContainer, padding 24px horizontal.

**Structure:**
```
VBoxContainer
  └─ Label (Heading 22px Bold #F5F5F5): "Poné el nombre de tu pibe"
  └─ LineEdit — pibe name (placeholder "Nombre del pibe", max_length=20)
  └─ Label (Display 28px Bold #F5F5F5) — live preview of typed name (updates on text_changed)
  └─ Separator (ColorRect 1px #3D3D3D, full width)
  └─ Label (Body 16px Regular #A0A0A0): "Stats base de tu pibe"
  └─ GridContainer (2 cols) — stat rows
       └─ Label stat name (14px Bold #A0A0A0): "Aguante"
       └─ Label stat value (14px Bold #F5F5F5): "50"
       └─ [repeat for each base stat]
  └─ Button (primary CTA, disabled until name ≥ 2 chars): "Así se llama mi pibe"
```

**Validation:**
- CTA disabled if name is empty or < 2 characters.
- Inline error below LineEdit if name is > 20 characters (blocked by max_length, but show hint): "Máximo 20 caracteres, chabón."
- Profanity/reserved name rejection (server-side): error "Ese nombre no va. Elegí otro." (Body 16px `#E67E22`)

**Base stats (Phase 1 fixed values — all pibes start equal):**
| Stat | Value |
|------|-------|
| Aguante | 50 |
| Velocidad | 50 |
| Astucia | 50 |
| Carisma | 50 |

### 5. TutorialScreen.tscn

**Purpose:** Bienvenida orientativa — single CTA, no branching.

**Layout:** Full-screen `#1A1A1A` → VBoxContainer centered, padding 32px horizontal.

**Structure:**
```
VBoxContainer (centered, spacing 24px)
  └─ Label (Heading 22px Bold #F5F5F5): "Bienvenido a la barra"
  └─ RichTextLabel (Body 16px Regular #F5F5F5, line-height 1.5):
       "Tu pibe acaba de llegar al barrio. Laburá duro, ganate el respeto de la hinchada,
        y llevá a tu club a lo más alto.

        Acá manda quien más se la banca."
  └─ [Spacer — flexible, pushes CTA down]
  └─ Button (primary CTA): "Dale, empezamos"
```

**No skip button. No pagination. Single screen.**

**CTA always enabled** (no pre-condition).

### 6. HomeScreen.tscn

**Purpose:** Main hub — Phase 1 version is a placeholder shell for Phase 2+ features.

**Layout:** Full-screen `#1A1A1A` → VBoxContainer. Bottom nav bar persistent.

**Structure:**
```
VBoxContainer
  └─ [Top bar — HBoxContainer]
       └─ Label (Heading 22px Bold #F5F5F5): pibe name (from save)
       └─ [Spacer]
       └─ Label (14px Bold #A0A0A0): club name
  └─ [Content area — ScrollContainer, flex-fill]
       └─ VBoxContainer (spacing 24px, padding 16px)
            └─ [Phase 1: empty state]
                 └─ VBoxContainer (centered)
                      └─ Label (Heading 22px Bold #F5F5F5): "Tu barra te espera."
                      └─ Label (Body 16px Regular #A0A0A0): "Empezá a laburar."
  └─ [Bottom nav bar — HBoxContainer, bg #2D2D2D, height 56px + safe-area inset]
       └─ NavButton: "Inicio" (active: #D62828 indicator dot above icon)
       └─ NavButton: "Barra" (placeholder)
       └─ NavButton: "Partidos" (placeholder)
       └─ NavButton: "Perfil" (placeholder)
```

**NavButton (custom scene):**
- VBoxContainer: [indicator dot ColorRect 4x4 `#D62828`, hidden when inactive] + [icon placeholder ColorRect 24x24 `#A0A0A0`] + [Label 14px Bold]
- Active label color: `#F5F5F5`; inactive: `#A0A0A0`
- Min touch width: 64px, height: 56px

**Bottom safe area:** Additional bottom padding = `DisplayServer.get_display_safe_area().position.y` equivalent for home indicator on iOS (runtime value, minimum 34px).

## Screen Flow

```
SplashScreen
    └─► AuthScreen (login/registro)
             └─► ClubPickerScreen (first-time only)
                      └─► PibeCreatorScreen (first-time only)
                               └─► TutorialScreen (first-time only)
                                        └─► HomeScreen (main loop)

Returning user: SplashScreen → AuthScreen (auto-login if token valid) → HomeScreen
```

All transitions: fade-out 150ms → fade-in 150ms.
