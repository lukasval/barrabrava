## Decisions Made

- **AI display names stored as `null` on server, rendered as canned label at UI layer (plan 03.04).** Avoids any real or LLM-generated name reaching storage (T-3-AS-04, D-14).
- **`ai_baseline.ts` kept as a separate module from `ai_seed.ts`.** Phase 6 balance pass = one-line PR to change constants, no risk of touching seeding logic.
- **`COL_TURNOS` omitted from client mirror.** Client never reads turnos directly — writes via `submit_turno` RPC, reads back via `get_roster`. Mirrors D-CR-01 (keep mirror tight).
- **`admin_force_recruit_refresh` is a Wave 1 stub.** Logs + audits + returns `{ok:true, stub:true}`. Real `runRecruitRefresh` lands in plan 03.03.
- **No second `registerLeaderboardReset` call.** The single Phase 2 line in `main.ts` covers all 4 leaderboard IDs via the dispatched `lb.id` switch in `onSchedulerLeaderboardReset`. Phase 2 hard-won lesson applied without regression.
