## Decisions Made

- **assign_profession lazy-commit order**: accrueIdleForPibe called BEFORE `pibe.profession =` switch. This is critical: if we switched first, the prior profession's earnings would be lost (no `last_collected_at` stamp = next collect sees 0 elapsed from old profession start).
- **commitAccrualForPibes exported**: allows plan 03.05 tests to call it directly for invariant testing (LAB-IDLE-IDEMPOTENT, LAB-SKILL-GRIND).
- **Tutorial bypass strategy**: `collect_idle {tutorial:true}` = fixed +10 Plata; `recruit_pibe {tutorial:true}` = free pick via `generatePick` without pool. Both preserve rank/rep validation except cost/rep_min. This avoids tutorial blocking on cron availability (first-time deploy before 05:00 ART).
- **submit_turno marker-first**: written unconditionally to `turnos/{fixture_id}`. Even if the Step 7 atomic batch fails due to concurrent write, the marker persists — next call returns the prior state via `idempotent_replay`. Client never needs to guess whether the turno landed.
- **elapsed_ms telemetry accepted risk (T-3-UIB-05)**: client can send any elapsed_ms value; server logs it for analytics but no business logic branches on it. Plan 03.05 LAB-TUTORIAL-DURATION test asserts `< 600000` (10 min) to catch obvious tampering in aggregate.
- **recruit_cron forClubId bypass**: when provided, the `generated_date_art` short-circuit is skipped for that one club. This allows admin to force-refresh a specific club without waiting until next day, and is the exact use case of `admin_force_recruit_refresh`.
- **electLideresForAllClubs as module-level function**: not nested inside `detectSeasonState` so it can be called independently and easily unit-tested in plan 03.05.
