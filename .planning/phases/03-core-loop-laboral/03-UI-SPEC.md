# Phase 3 — UI Design Contract: Core Loop Laboral

> Visual and interaction contract for BarraBrava Phase 3 (Core Loop Laboral). Generated by gsd-ui-researcher.
> Platform: Mobile (iOS + Android). Godot 4.3. Canvas 1080x1920, min viewport 360px width, portrait only.
> **Inherits Phase 1 UI-SPEC.** This document only declares **additions**, **extensions**, and **phase-specific overrides**. Anything not respec'd here defers to Phase 1.

## 3b. Typography — Phase 3 Scale (4 sizes, hard cap honored)

Phase 3 keeps the 4-size scale. The Heading role moves from 22 (Phase 1) to **20** (Phase 3), and that same 20 also serves as the Numeric Counter for ResourceWidget. Both font weights already loaded by Phase 1 Theme.tres.

| Role | Size | Weight | Line Height | Godot Node | Usage |
|------|------|--------|-------------|------------|-------|
| Display | 28px | Bold 700 | 1.1 | Label | Pibe name in PibeDetailScreen hero; live name preview |
| **Heading / Numeric Counter** | **20px** | **Bold 700** | **1.2 (Heading) / 1.0 (Numeric)** | **Label** | **Dual role.** Heading: screen titles ("Mis pibes", "Reclutar pibes", "Aguantadero Nv. {N}", "Asignar profesión", "Hacer turno", "Habilidades", "Subir a nivel {N+1}", "Bandera room", "Bienvenido a la barra", tutorial step titles). Numeric Counter: ResourceWidget numeric value (e.g. "1.250"). Same size + weight; line-height differs by context (1.2 for paragraph-style titles, 1.0 for compact numerics inside 80x60 widgets). |
| Body | 16px | Regular 400 | 1.5 | RichTextLabel | descriptions, empty states |
| Label / Button | 14px | Bold 700 | 1.2 | Button / Label | chip labels (RankBadge, TraitChip, SortChip), button text, helper labels |

**Total declared sizes: 4 (14 / 16 / 20 / 28).** No widget-only or chip-only typography overrides exist. Every text element on every screen in Phase 3 resolves to exactly one of these four sizes.

Rules carry from Phase 1 (no italic / no underline / no mixed weights inside a Label).

**Number formatting:** Spanish locale thousands separator = `.` (e.g. `1.250` not `1,250`). Decimals = `,`. Rates shown as `+10 Plata/h` (no decimals for whole numbers). Implementation note for executor: use `"%d".format()` then post-process or `String.num_int64()` + manual separator.

## 4. Color — Phase 3 Additions

Carries Phase 1 palette verbatim. Adds **3 semantic-domain palettes** for new game systems: **rank**, **profession**, and **trait sentiment**. Each is additive — none overrides the 60/30/10 dominant/secondary/accent split.

### 4.1 Accent reservation — extended list

Phase 1 reserved accent `#D62828` for 5 elements. Phase 3 adds:

6. **"Hacer turno" CTA** on HomeScreen when `window.state in ["open", "live"]` — same red as primary CTA, with pulse animation (see §10)
7. **RecruitCard reveal-flash** when 2nd trait is unveiled post-recruit — single 600ms flash on the new TraitChip
8. **RankBadge active border** when player just promoted (one-time 1.2s flash on HomeScreen return after Rep crosses threshold)

Accent remains forbidden on: nav bar background, regular text, card backgrounds, bottom nav non-active items, ResourceWidget numbers.

### 4.2 Rank colors (NEW semantic palette)

Used only on `RankBadge` component. Five ranks, five colors. Chosen for ladder-readability + accessibility (4.5:1 contrast against `#2D2D2D` card backgrounds).

| Rank | Hex | Godot Color | Semantic |
|------|-----|------------|----------|
| Pibe | `#9CA3AF` | `Color(0.612, 0.639, 0.686, 1)` | Neutral gray — starter |
| Soldado | `#3B82F6` | `Color(0.231, 0.510, 0.965, 1)` | Blue — rank-up step 1 |
| Capo | `#A855F7` | `Color(0.659, 0.333, 0.969, 1)` | Purple — rank-up step 2 |
| Mesa Chica | `#F59E0B` | `Color(0.961, 0.620, 0.043, 1)` | Amber — top 5 of club |
| Líder | `#FBBF24` | `Color(0.984, 0.749, 0.141, 1)` | Gold — apex |

**Constraint:** rank colors NEVER substitute for accent on CTAs. They appear only on `RankBadge` background fill.

### 4.3 Profession colors (NEW semantic palette)

Used only on `ProfessionIcon` background tint chip. 5 professions + 1 Líder-only.

| Profession | Hex | Lunfardo label |
|------------|-----|----------------|
| Trapito | `#06B6D4` | "Trapito" (cyan — calle) |
| Vendedor | `#22C55E` | "Vendedor" (green — produce) |
| Patovica | `#EF4444` | "Patovica" (red-orange — fuerza) |
| Remisero | `#EAB308` | "Remisero" (yellow — auto) |
| Hablar cana (Líder-only) | `#94A3B8` | "Hablar cana" (steel — neutral) |
| Sin profesión | `#3D3D3D` | "Sin laburo" (border-inactive gray) |

Vendedor sub-types (chori / paty / entradas / merch / bengalas) share the green tint; sub-type label is text-only inside the icon chip. No new color per sub-type to avoid palette bloat.

### 4.4 Trait sentiment colors (NEW semantic palette)

Used only on `TraitChip`. Each of the ~14 traits is pre-classified positive / negative / neutral by content (see Trait Pool §8.4). Sentiment is the chip border, not background — keeps secondary surface uniform.

| Sentiment | Border color | Hex | Examples |
|-----------|--------------|-----|----------|
| Positive | Soft green | `#10B981` | Aguantador, Pícaro, Veterano |
| Negative | Soft amber | `#F97316` | Pies de plomo, Buchón, Cabezón |
| Neutral | Border-inactive | `#3D3D3D` | Cordobés, Porteño, Pendejo |

**Sentiment is descriptive, not gameplay-decisive in Phase 3.** Traits modify nothing mechanically in Phase 3 (they flavor Phase 4 combat + Phase 5 Cronista). UI uses sentiment color purely as readability cue, with text label always present so colorblind users get the same info.

### 4.5 Energía bar gradient (state-shifted, NOT a gradient fill)

`EnergiaBar` is a horizontal progress bar (Phase 1's flat-fill rule still applies — no actual gradient). The fill color **shifts at thresholds**:

| Energía range | Hex | Meaning |
|---------------|-----|---------|
| 70-100 | `#22C55E` | Full — green |
| 30-69 | `#EAB308` | Mid — yellow |
| 1-29 | `#F97316` | Low — amber (matches negative-trait border) |
| 0 | `#A0A0A0` | Empty — text secondary gray, with label "Descansando" |

Color shifts at integer thresholds, no interpolation. Bar background track: `#2D2D2D` (secondary).

### 4.6 Resource icon tints

| Resource | Hex | Token in AppTheme.gd |
|----------|-----|----------------------|
| Plata | `#FBBF24` | `RES_PLATA` (gold) |
| Aguante grupal | `#D62828` | `RES_AGUANTE` (accent red — group identity) |
| Reputación | `#A855F7` | `RES_REPUTACION` (purple) |
| Visto Bueno Cana | `#94A3B8` | `RES_VBC` (steel gray) |

**Aguante shares accent red.** Justified: Aguante is the club's collective resource — the same color as the brand accent reinforces "this is the barra's lifeblood." Plata/Rep/VBC stay off-accent.

**Resource disambiguation (post-typography-fix):** With the ResourceWidget secondary label removed (see §6.3), each resource is identified by (a) its unique icon tint above, and (b) the numeric value rendered in Numeric Counter 20 Bold. While custom icon art is pending, a one-letter glyph in the icon slot (`$` Plata / `A` Aguante / `R` Reputación / `V` VBC) at 16px Bold acts as placeholder. Executor swaps to SVG art when assets land.

## 10. Tutorial State Machine

**Flag:** `players.profile.tutorial_done: bool` (server-side, set true on completion or skip).

**Entry condition:** Post-PibeCreator success, if `tutorial_done == false`, FlowRouter pushes TutorialScreen before HomeScreen. Returning user with `tutorial_done == true` skips straight to HomeScreen.

**6 scripted steps:**

| # | Step ID | Title | Body copy | Illustration | CTA label |
|---|---------|-------|-----------|--------------|-----------|
| 1 | `welcome` | "Bienvenido a la barra" | "Tu pibé llegó al barrio. Te vamos a mostrar el laburo en 5 pasos." | Pibe avatar centered, club crest behind | "Dale, vamos" |
| 2 | `recruit_intro` | "Sumá tu primer pibé" | "Cada día llegan 3 pibes nuevos al barrio. Elegí uno para sumarse a tu barra." | RecruitScreen mock-up screenshot | "Ir a reclutar" |
| 3 | `assign_profession` | "Ponelo a laburar" | "Tu pibé puede laburar de trapito, vendedor, patovica o remisero. Genera Plata mientras vos no estás." | ProfessionAssignScreen mock-up | "Asignar profesión" |
| 4 | `see_plata` | "Cobrá la Plata" | "Cada hora que tu pibé labura, suma Plata al tope. Cobrala cuando vuelvas. Cap: 12 horas." | ResourceWidget Plata animation | "Entendido" |
| 5 | `first_turno` | "Hacé tu primer turno" | "Cuando tu club juega, abrí la ventana y mandá a tus pibes al estadio. Eso le suma Aguante al pozo grupal." | HomeScreen with TurnoButton active + window banner | "Hacer turno (simulado)" |
| 6 | `reward` | "Bienvenida con regalo" | "Para arrancar bien, te damos tu primer trapo y un cántico. Aplaudila al equipo." | Trapo placeholder + sound icon | "Empezar de verdad" |

**Transitions:**
- step `welcome` → step `recruit_intro` (only forward; back hidden on step 1)
- All other steps support `Atrás` button → previous step
- `Saltar tutorial` on any step → ConfirmationDialog → if confirm: server RPC `mark_tutorial_done` + go HomeScreen
- step `reward` CTA → server RPC `grant_tutorial_reward` (credits first trapo + cántico cosmetic flag) + set `tutorial_done = true` + go HomeScreen
- step 5's CTA "Hacer turno (simulado)" does NOT submit a real turno. It runs a client-side animation showing the modal opening + a fake +50 Aguante toast, then advances to step 6. No RPC.

**Idempotency:** If user kills app between steps, `tutorial_done` stays false until step 6 CTA. They resume at step 1 on relaunch. Phase 3 does not persist intermediate step. (Future: persist step index in player profile if analytics shows drop-off mid-tutorial.)

**Time budget:** Target <10 min total (ONB-05). Steps 1, 3, 4, 6 are read-and-tap (~30s each). Steps 2, 5 require small interactions (~2 min each).

## 12. Accessibility

| Requirement | Implementation |

| Font size | Phase 3 uses exactly the 4-size scale from §3b (14 / 16 / 20 / 28). No widget-only or chip-only overrides exist. Minimum on-screen text is 14px, satisfying mobile readability heuristics. |
