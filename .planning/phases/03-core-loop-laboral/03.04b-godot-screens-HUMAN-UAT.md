## Walkthrough A — Fresh User Flow (target: <10 min, ONB-05)

**What is being tested:** Full onboarding path from AuthScreen through 6-step tutorial to HomeScreen. Verifies ONB-05 (<10 min) and ONB-06 (reward at step 6).

**Steps:**

1. Launch from AuthScreen. Register new account: `test+p3client@example.com / testpass1234`. Accept terms.
2. ClubPickerScreen: pick any Primera club (e.g., "Xeneizes" parody of Boca). Confirm faccion.
3. PibeCreatorScreen: name pibe, confirm.
4. **TutorialScreen appears** (ONB-05 gate via FlowRouter.go_post_pibe_create). Walk through all 6 steps:
   - Step 1 "Bienvenido a la barra": tap "Dale, vamos".
   - Step 2 "Sumá tu primer pibé": tap "Ir a reclutar" (acknowledgement only, no actual recruit mid-tutorial).
   - Step 3 "Ponelo a laburar": tap "Asignar profesión" (acknowledgement only).
   - Step 4 "Cobrá la Plata": tap "Entendido" (triggers collect_idle tutorial bypass → +10 Plata).
   - Step 5 "Hacé tu primer turno": tap "Hacer turno (simulado)" (acknowledgement; server credits +20 Rep via tutorial bypass).
   - Step 6 "Bienvenida con regalo": tap "Empezar de verdad" → routes to HomeScreen.
   - Verify: skip button at any step surfaces ConfirmationDialog "¿Saltar el tutorial? Vas a perderte la recompensa de bienvenida." with "Sí, saltar" / "Seguir".
5. HomeScreen: confirm:
   - ResourceRow shows 4 widgets: Plata ~10 (from step 4 collect_idle bypass), Aguante 0, Rep ~20 (from step 5 bypass), VBC 0.
   - RankBadge = "Pibe" (gray).
   - FaccionLabel + ClubName render in TopBar.
   - WindowBanner shows Phase 2 state ("Falta HH:MM" or "Ventana abierta").
   - BottomNav = Inicio (active) / Roster / Aguantadero / Reclutar.
   - QuickActions row has 3 buttons: "Laburar", "Reclutar", "Aguantadero".
6. Tap "Reclutar". RecruitScreen:
   - 3 RecruitCard render: trait_1 visible, trait_2 shows "?".
   - RankGateLabel: "Tu rango: Pibe — costo 500 Plata, máx 2 pibes".
   - Refresh countdown shows "Próxima ronda en HH:MM".
   - Tap first card → ConfirmationDialog "Reclutar a {nombre}? Te cuesta 500 Plata."
   - Confirm → expect lunfardo error "Te faltan 490 de Plata." (since only ~10 Plata after tutorial) OR card shows disabled state with that copy pre-tap.
7. Use admin curl to grant 600 Plata via Nakama storage console or `admin_grant_rep`-style edit. Refresh HomeScreen.
8. Re-attempt recruit. Confirm. Card disappears from pool. Route to RosterScreen — verify new pibe entry visible.
9. Open PibeDetailScreen via RosterScreen tap:
   - Hero section: 120×120 avatar placeholder + Name (Display 28 Bold) + RolLabel.
   - Both trait chips revealed (now post-recruit).
   - EnergiaBar at 100, label "100/100 — Regenera +5/h".
   - No skills yet (skills section empty or shows "Ningún laburo asignado aún").
   - CurrentAssignment shows "Descansando".
   - Actions row: "Asignar profesión" (primary), "Enviar a turno" (secondary, disabled — no live window).
10. Tap "Asignar profesión". ProfessionAssignScreen:
    - 5 rows visible (Trapito, Vendedor, Patovica, Remisero, + 1 more per spec).
    - Hablar cana row visible-but-disabled (modulate 0.4) + gate label "Esta profesión es solo para el Líder de la barra."
    - Tap "Trapito" row → ConfirmationDialog "Asignar a {nombre} como Trapito? Empieza a generar Plata ya." → confirm → success.
11. Return to HomeScreen. After a few minutes, IdleNotice appears: "Tenés X de Plata sin cobrar." (or trigger via admin).

**Expected result:** Full flow completes without crashes. Tutorial finishes in under 10 minutes. Reward (trapo placeholder + cántico unlock confirmation) visible after step 6.

**Steps:**

2. AguantaderoScreen renders:

   - StatsRow: "Capacidad {cap}/20", "Aguante pasivo +0/h (deferred)", "Próx nivel {cost} Plata".

**Expected result:** Upgrade flow completes without crashes. BanderaRoom slot 1 shows tutorial trapo. Level and capacity update correctly after upgrade.

## Walkthrough C — Turno Modal (~2 min, requires open window)

**What is being tested:** TurnoButton appears when window is open/live, TurnoModal multi-select + aporte estimado work, idempotent re-submit does not double-credit.

**Steps:**

1. If no live window: use `admin_inject_test_fixture` RPC (Phase 2 admin RPC) to create one.
   ```bash
   curl -fsS -X POST "https://nakama-production-7ea8.up.railway.app/v2/rpc/admin_inject_test_fixture" \
     -H "Authorization: Bearer $ADMIN_BEARER" \
     -H "Content-Type: application/json" \
     -d '{"http_key":"defaulthttpkey","unwrap":true}'
   ```
2. HomeScreen: TurnoButton appears with pulse animation (A-03). Button text: "Hacer turno" when eligible pibes exist, or "Sin pibes con energía" when none qualify.
3. Tap TurnoButton. TurnoModal slides up:
   - Lists eligible pibes (energía ≥ 30, not already in turno).
   - Checkboxes for each pibe.
   - Aporte estimado label updates as you check/uncheck: "+50 Aguante × N · +20 Rep × N".
4. Select at least one pibe. Tap confirm. Energy drain animation (A-06) plays. Modal closes.
5. HomeScreen ResourceWidget Rep increments (or toast appears with "+X Rep").
6. WindowBanner updates to "X pibes en turno." (or equivalent state copy).
7. Re-tap TurnoButton. Try to submit same pibes again with same fixture → server returns idempotent replay → UI does NOT show "+50 Aguante" toast a second time. Modal closes cleanly.

**Expected result:** TurnoModal opens/closes cleanly. Aporte estimado label updates in real time. Idempotent re-attempt confirmed — no double-credit toast.

**What is being tested:** App-Store gate (UI-SPEC §8.5 blacklist), 4-size typography compliance (14/16/20/28), color palette compliance, lunfardo ConfirmationDialog copy.

**Steps:**

1. Open every screen (HomeScreen, RosterScreen, RecruitScreen, PibeDetailScreen, ProfessionAssignScreen, AguantaderoScreen, TutorialScreen). Scroll all visible text.
2. Confirm ZERO occurrences of banned vocabulary in any visible copy: "ataque", "raid", "pelea", "atacar", "robar".
3. Confirm typography uses only 4 sizes: 14px, 16px, 20px, 28px. No 22px anywhere.
4. Confirm color palettes:

**Expected result:** No banned vocabulary visible anywhere. All text uses 4-size scale only. Color palettes match spec. ConfirmationDialogs use ES copy.

## Walkthrough E — Error States + Tutorial Telemetry (~3 min)

**What is being tested:** Network error states render with lunfardo copy, roster-cap error triggers correct copy, hablar_cana rank gate enforced, tutorial_duration_ms server log line present.

**Steps:**

1. **Network disconnection test:** Disconnect WiFi mid-action (e.g., during RosterScreen load or RecruitScreen load). Expect:
   - Error state renders with copy "Sin conexión." (or equivalent lunfardo).
   - Retry button visible and functional (reconnect WiFi, tap retry → screen reloads).
2. **Roster cap error:** Recruit 2 pibes (total = 2 at Pibe rank). Attempt to recruit a 3rd. Expect: cap-reached banner "Tu aguantadero está lleno ({cap}/{cap}). Subí de nivel para sumar más pibes." OR error copy "Llegaste al tope de pibes para tu rango. Subí de Soldado primero."
3. **Hablar cana rank gate:** In ProfessionAssignScreen, verify hablar_cana row is visible-but-disabled when rank is not "lider". Tap disabled row — should show gate copy "Esta profesión es solo para el Líder de la barra." (not crash, not silent dismiss).
4. **Tutorial telemetry verification:** SSH/tail Railway logs (or use Railway dashboard log stream) while a fresh test user completes the tutorial. After step 6 completes, expected log line:
   ```
   [complete_tutorial] user=<id> tutorial_done=true tutorial_duration_ms=<N>
   ```
   Sanity check: N should be in the 60000–600000 ms range (1–10 min). N < 600000 confirms ONB-05 target (LAB-TUTORIAL-DURATION invariant precursor for plan 03.05).

**Expected result:** All error states render with lunfardo copy + retry buttons. Roster cap enforced with correct copy. Hablar cana gate copy shows for non-Líder. Server log contains `tutorial_duration_ms` with plausible value < 600000.
